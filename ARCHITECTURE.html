🔐 Azure SSO Login Flow

1
User clicks "Sign in with Microsoft"
2
MSAL loginRedirect() → Microsoft Entra ID (prompt: select_account)
3
User authenticates at login.microsoftonline.com
4
Microsoft redirects to localhost:5173/login with auth code
5
handleRedirectPromise() exchanges code → id_token + access_token
6
POST /auth/azure with id_token + access_token
7
Backend decodes JWT, calls Graph API for manager + groups
8
Upsert user, sync role from AD groups, issue internal JWT
9
Frontend redirects to /{role} dashboard

🎯 Goal Creation → Approval Flow

1
Employee creates goal sheet (DRAFT) for active cycle
2
Adds goals: ≥10% each, ≤8 goals, must total 100%
3
Employee submits → status: SUBMITTED, goals editable
4
SendGrid email → manager notified
5
Manager reviews, optionally inline-edits targets/weightage
6a
Approve → status: APPROVED, all goals is_locked=True
6b
Return → status: RETURNED, goals unlocked, reason stored
7
SendGrid email → employee notified of outcome

📊 Quarterly Check-in Flow

1
Cycle window opens (Q1/Q2/Q3/Q4)
2
Employee logs actuals: actual_numeric/date + status per goal
3
Score computed: Min=actual÷target, Max=target÷actual, Zero=1 if 0
4
Manager views planned vs actual across team
5
Manager adds check-in comment (structured feedback)
6
Analytics: QoQ trends, heatmaps updated in real-time

⚠️ Escalation Engine Flow

1
APScheduler triggers run_all_checks() every 6 hours
2
Check: employee not submitted within threshold_days
3
Check: manager not approved within threshold_days
4
Check: check-in not completed within window
5
On breach: create EscalationEvent record
6
Walk notify_chain (employee → manager → HR)
7
Send email via SendGrid per chain member
8
Admin sees open/resolved events in dashboard
